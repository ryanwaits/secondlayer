The crux discovery: raw events carry the stacks-node's verbose serde-tagged Clarity JSON ({UInt:223}, {Tuple:{Sequence:{String:{ASCII…}}}}) that Index only stores as hex. The golden-diff caught every leak — nft tokenId, print value, the contract_call event spread, even empty memo. All now decode from the canonical hex (raw_value / function_args_hex), making the two sources byte-identical and cleaner for handler authors.

§6

Risk Map & Tracked Gaps

1
SDK duplicates the Index/Streams wire types
The public SDK hand-maintains IndexEvent/IndexTransaction shapes that now also live in @secondlayer/shared/index-http. Two copies can drift.
→ FT.1 (tracked): converge SDK onto the shared transport types; deferred (touches SDK public types) until the shapes drift.
2
Streams rate-limit is process-local
streams/rate-limit.ts uses an in-process sliding window — correct only for a single API instance. TODO notes Redis before scaling out.
→ move to Redis-backed limiting before running multiple API replicas.
3
Index reorgs stubbed on tx-level endpoints
/contract-calls + /transactions return reorgs: [] by design — Streams is the reorg authority. Not a bug; documented in-code.
→ by design: consumers (incl. the subgraph runtime) take reorg signal from Streams.
4
Direct chain-level subscriptions (#8) deferred
The triggers/ stub (EventTrigger{filter}, 13 typed on.* constructors) is inert — not wired to the emitter or API. Subscriptions still require a subgraph.
→ deferred in favor of #7 (the re-point). Delivery stack is already subgraph-agnostic; a second outbox producer would unlock it.
5
Value normalization is a behavior change
nft tokenId / print value / contract_call spread now decode from hex (clean) instead of the node serde form. Changes handler output for those types.
→ ~0 blast radius (only 3 internal benches deployed, none nft/print external); reindex affected subgraphs. Changeset documents it.
6
Index mempool is a single-node, go-forward view
Built from one node's observer; no pre-observer backlog replay. Fine for tx-lifecycle/optimistic-UX, not global mempool/MEV aggregation.
→ by design (documented). No standard bulk-mempool RPC exists to aggregate.
No critical issues. Every gap is either tracked tech-debt (1–2) or an intentional, documented design choice (3–6). The re-point shipped with full golden-diff parity and a live, instantly-reversible prod canary.

Decision Log

tooling on services
Subgraphs consume Streams (clock) + Index (data) over HTTP — live the Kourier "build on the public microservices" model, realized.
Streams = reorg authority
Reorg detection lives at Streams; Index tx-level endpoints stub reorgs:[]; subgraph runtime polls /reorgs.
Index = data, registry = platform
Trait resolution reads the contract registry on targetDb (platform DB the processor always holds) → source-independent, no HTTP trait endpoint needed.
decode from canonical hex
nft/print/contract_call values decode from raw_value/function_args_hex, not the node's serde-tagged JSON (unreproducible from Index).
transport in shared leaf
@secondlayer/shared/index-http — both SDK and subgraphs depend on shared; avoids the sdk→subgraphs cycle.
self-host scope
Fully-self-hosted only (own api + indexer + registry + processor). No public-API-consumer self-host.
GraphQL
Deliberate NO — REST + typed ORM codegen is the answer across all surfaces.
#8 direct subscriptions
Deferred (separate effort). Subscriptions sit on subgraph rows (L3) / decoded data — not raw L1 (Chainhooks' lane).
rollout posture
Per-subgraph eligibility gate; default db; SUBGRAPH_SOURCE=streams-index committed to prod compose (survives git reset --hard deploys). Unset = instant rollback.
