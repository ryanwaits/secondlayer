How it works

Never write another ETL pipeline.

Three files, one deploy. Sources declare what to listen for. Schema defines the tables. Handlers transform each event into rows. We do the rest.

01 / SOURCES

Subscribe to onchain events.

Point at contracts, filter by event type, fork at any block. Stacks mainnet, testnet, or a local devnet — same config. 

sources: {
  transfer: {
    type: "stx_transfer"
  }
}
02 / SCHEMA

Typed Postgres, no migrations.

Declare columns like you would in TypeScript. Clarity types map cleanly. Indexes declared inline. We migrate on deploy. 

schema: {
  transfers: {
    sender: { type: "principal" },
    amount: { type: "uint" }
  }
}
03 / HANDLERS

Transform events into rows.

A handler is a function. Receives a typed event, writes typed rows. Block-level atomic. Reorg-safe. Idempotent by design. 

handlers: {
  transfer: (event, ctx) => {
    ctx.insert("transfers", event)
  }
}

defineSubgraph

Your whole indexer, in one file.

No schema.graphql. No separate entity files. No migration runner. Just one typed function call. We infer the rest.

Questions

Things Indexers actually ask.

What chains are supported?
Stacks mainnet and testnet today. Bitcoin L1 inclusion events are piggy-backed via Stacks block metadata. Additional L2s shipping on the Second Layer roadmap — follow the changelog.
How does pricing scale?
You're billed on indexed events and storage, metered hourly. No seat fees, no per-query cost. Overages are transparent — the dashboard shows projected spend before the month closes.
Can I self-host?
Yes. The runtime is MIT-licensed. Ship a docker image, point at any Stacks node, bring your own Postgres 15+. Same defineSubgraph config runs managed or self-hosted — no forks.
What's the latency from block to query?
Handlers run in-process with the block stream. Median block-to-row latency is under 800ms on managed infra. Your query sees the row the moment the block commits.
What happens on a reorg?
Blocks write inside a transaction tagged with the block hash. On a reorg we revert all writes for the orphaned blocks and replay the canonical chain. Your handlers stay pure functions — no manual cleanup.
