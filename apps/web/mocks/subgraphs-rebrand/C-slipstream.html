How it works

Three primitives. Everything else is just SQL.

Declare what to listen to, what tables to create, and how each event becomes rows. The runtime handles reorgs, backfills, and downstream deltas.

Declarative sources.

Name the events you care about. STX transfers, FT, NFT, contract calls, or a specific print topic.

sources: {
  transfer: { type: "stx_transfer" }
}

Strict schema.

Columns have Clarity-native types: principal, uint, buff, tuple. Indexed flags propagate to Postgres.

schema: {
  transfers: {
    sender: { type: "principal" }
  }
}

Block-level atomicity.

Handlers transform each event into rows. Every insert is atomic per block — reorgs roll back cleanly.

handlers: {
  transfer: (event, ctx) => {
    ctx.insert("transfers", ...)
  }
}

Define

One file. One deploy.

Subgraphs are just TypeScript. Sources, schema, handlers — fully typed end-to-end. No YAML. No schema migrations to write by hand.

stx-transfers.ts
import { defineSubgraph } from "@secondlayer/subgraphs" export default defineSubgraph({ name: "stx-transfers", sources: { transfer: { type: "stx_transfer" }, }, schema: { transfers: { columns: { sender: { type: "principal", indexed: true }, recipient: { type: "principal", indexed: true }, amount: { type: "uint" }, memo: { type: "text", nullable: true }, }, }, }, handlers: { transfer: (event, ctx) => { ctx.insert("transfers", { sender: event.sender, recipient: event.recipient, amount: event.amount, memo: event.memo, }) }, }, })
