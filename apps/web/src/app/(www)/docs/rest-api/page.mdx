# REST API

The shared envelope, cursor, and filter grammar across the read surfaces ‚Äî datasets, index, streams, and subgraph tables.

## Response envelope

Datasets and index reads come back in a `data` array; `meta` carries the total and a stable cursor.

```json
{
  "data": [{ "_id": 48138, "block_height": 8054704 }],
  "meta": { "total": 48138, "cursor": "eyJpZCI6NDgxMzd9" }
}
```

`/v1/subgraphs` speaks a cursor envelope instead ‚Äî `rows`, a resumable `next_cursor`, and the indexing `tip`:

```json
{
  "rows": [{ "_id": 48138, "block_height": 8054704 }],
  "next_cursor": "48137",
  "tip": { "block_height": 8054704, "subgraph_height": 8054704, "blocks_behind": 0 }
}
```

## Pagination

- `_limit` ‚Äî page size (default 10).
- `_order` ‚Äî `asc` or `desc`.
- `_cursor` ‚Äî opaque token from the previous response's `meta.cursor` (data/meta surfaces).
- `cursor` ‚Äî on `/v1/subgraphs`, pass the previous response's `next_cursor` to resume (`_id` keyset).

## Filtering

Filter on any column by passing it as a query param. The filter grammar is identical across every surface.

```bash
curl https://api.secondlayer.tools/v1/subgraphs/sbtc-flows/transfers \
  -G \
  -d "sender=SP3PE7Q9...X44J" \
  -d "_order=desc"
```

## /v1 subgraphs open reads

Public subgraphs are anon-readable on `/v1/subgraphs` ‚Äî wildcard CORS, no key. Private subgraphs need the owning account's `sk-sl_` bearer; anonymous requests get `404` (no existence leak).

- `GET /v1/subgraphs` ‚Äî list public subgraphs (plus your own with a bearer).
- `GET /v1/subgraphs/:name` ‚Äî metadata: tables, columns, indexing tip.
- `GET /v1/subgraphs/:name/:table` ‚Äî rows; `/count`, `/aggregate`, `/:id`, `/stream` (SSE) alongside.

Pagination is `_id`-keyset: `?cursor=<next_cursor>` resumes, `_order=asc|desc` sets direction. `_offset` and `_sort` are **rejected with `400`** on `/v1` ‚Äî they live only on the authed `/api/subgraphs` dashboard surface. `_limit`, `_fields`, `_search`, and column filters work as everywhere else.

Rate limits: anonymous 100 rps, keyed 50 rps ‚Äî watch the `X-RateLimit-*` headers.

## Streaming (SSE)

Two endpoints push rows in real time as Server-Sent Events (`Content-Type: text/event-stream`) over a single long-lived connection. Both emit a `ping` keepalive every 20s.

- `GET /v1/streams/events/stream` ‚Äî the raw chain event firehose. Bearer auth. Accepts the same filters as `/v1/streams/events` (`types`, `not_types`, `contract_id`, `sender`, `recipient`, `asset_identifier`) and a start position (`cursor`/`from_cursor`/`from_height`); with no start position it live-tails from the reorg-clamped tip.
- `GET /v1/subgraphs/:name/:table/stream` ‚Äî new rows for a subgraph table. Anon for public subgraphs, `sk-sl_` bearer for your private ones. Accepts `?since=<block_height>` (replay from a height, then tail) plus the same column filters as the table's REST read.

```bash
curl -N -H "Authorization: Bearer $SL_API_KEY" \
  "https://api.secondlayer.tools/v1/streams/events/stream?types=ft_transfer"
```

See [Streams](/docs/streams#real-time-firehose-sse) and the [SDK](/docs/sdk#subscribe-to-the-live-stream-sse) for ergonomic clients.

## Aggregates

`GET /v1/subgraphs/:name/:table/aggregate` runs scalar aggregates over the same filtered set as the list/count endpoints ‚Äî anon for public subgraphs, `sk-sl_` bearer for your private ones. Pass any of the column filters above plus aggregate control params:

- `_count=true` ‚Äî `COUNT(*)`.
- `_countDistinct=col1,col2` ‚Äî distinct count per column.
- `_sum=col`, `_min=col`, `_max=col` ‚Äî comma-separated columns; `_sum/_min/_max` accept **numeric columns only** (`uint`/`int`, plus the system `_block_height`).

```bash
curl https://api.secondlayer.tools/v1/subgraphs/sbtc-flows/transfers/aggregate \
  -G \
  -d "status=active" \
  -d "_count=true" \
  -d "_sum=amount" \
  -d "_min=amount" \
  -d "_countDistinct=sender"
```

```json
{
  "count": 42,
  "countDistinct": { "sender": 7 },
  "sum": { "amount": "12300000000" },
  "min": { "amount": "1000000" }
}
```

- With no aggregate param the response is `{ "count": <n> }`.
- `count` and `countDistinct.col` are JSON numbers. `sum`/`min`/`max` are **lossless strings** (the underlying `NUMERIC`/`BIGINT` cast to text so large values never lose precision). `sum` over an empty set is `"0"`; `min`/`max` are `null` over an empty/all-null set.
- `400 NON_NUMERIC_COLUMN` for a non-numeric `_sum/_min/_max` column, `400 INVALID_COLUMN` for an unknown column, `400 TOO_MANY_AGGREGATES` past 32 aggregate columns per request.

The [SDK](/docs/sdk) exposes this as a typed `client.aggregate(spec)` whose result shape is inferred from the spec.

The `/api/subgraphs/*` equivalents of these endpoints remain as the authed dashboard/control-plane surface (session or key).

## Transaction-inclusion proof

`GET /v1/index/transactions/:tx_id/proof` returns a trustless inclusion proof: the raw transaction, raw header, and merkle path needed to verify, client-side, that a transaction is in a Stacks (Nakamoto) block ‚Äî plus the reward set, when resolvable, to confirm the signers attested to it.

```bash
curl "https://api.secondlayer.tools/v1/index/transactions/0x<tx_id>/proof"
```

```json
{
  "txid": "<hex>",
  "index_block_hash": "<hex>",
  "block_height": 8199502,
  "tx_index": 0,
  "raw_tx": "<hex>",
  "raw_header": "<hex>",
  "tx_merkle_path": [{ "position": "left", "hash": "<hex>" }],
  "consensus": {
    "reward_cycle": 136,
    "reward_set": { "signers": [{ "signing_key": "<hex>", "weight": 51 }], "total_weight": 3862 }
  }
}
```

- `consensus` is present only when the reward set could be resolved; otherwise the proof is anchored-only.
- `404 { error, code: "PROOF_UNAVAILABLE" }` ‚Äî the transaction or its block was not found.
- `503 { error, code: "PROOF_TX_SET_INCOMPLETE" }` ‚Äî the server could not reproduce the block's `tx_merkle_root` from its stored transaction set, so it refuses to emit a proof that wouldn't verify (fail-safe).
- Open during beta ‚Äî no key required, like the other read endpoints.

See [Verification](/docs/verification) for the trust levels and the `@secondlayer/sdk` verifier.
