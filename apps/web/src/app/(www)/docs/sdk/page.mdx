# SDK

`@secondlayer/sdk` ‚Äî a typed TypeScript client for every Secondlayer surface: Streams, Index, Subgraphs, and Subscriptions, plus webhook verification and a checkpointed consumer for indexers.

## Install

```bash
bun add @secondlayer/sdk
```

## Create a client

```ts
import { SecondLayer } from "@secondlayer/sdk";

const sl = new SecondLayer({
  apiKey: process.env.SL_API_KEY, // or a session token
  baseUrl: "https://api.secondlayer.tools", // default
});
```

## One client, every surface

- `sl.streams` ‚Äî raw, ordered L1 chain events (cursor-paginated, replayable).
- `sl.index` ‚Äî the decoded L2 layer: FT/NFT transfers, all event types, and contract calls.
- `sl.datasets` ‚Äî Foundation Datasets (sBTC, BNS, PoX-4, STX transfers) + catalog.
- `sl.contracts` ‚Äî find deployed contracts by trait (SIP-009/010/013).
- `sl.subgraphs` ‚Äî your app-specific L3 tables.
- `sl.subscriptions` ‚Äî create and manage webhook subscriptions (subgraph rows or raw chain events).

```ts
const tip = await sl.streams.tip();
const page = await sl.streams.events.list({
  types: ["ft_transfer"],
  contractId: "SP000000000000000000002Q6VF78.sbtc-token",
  limit: 10,
});
```

## Subscribe to chain events

`sl.subscriptions` creates webhook subscriptions. Beyond binding to a subgraph table, you can subscribe directly to **raw chain events** with no subgraph at all ‚Äî pass a `triggers` array built with the `trigger.*` factories (imported from the SDK root):

```ts
import { trigger } from "@secondlayer/sdk";

await sl.subscriptions.create({
  name: "amm-swaps",
  url: "https://my-app.com/webhook",
  triggers: [
    trigger.contractCall({ contractId: "SP....amm", functionName: "swap-*" }),
    trigger.ftTransfer({ trait: "sip-010", minAmount: "1000000" }),
  ],
});
```

Builders exist for every event type (`contractCall`, `contractDeploy`, `printEvent`, and the `stx*`/`ft*`/`nft*` transfer/mint/burn variants). See [Subscriptions](/docs/subscriptions#direct-chain-subscriptions-no-subgraph) for the full trigger catalog and delivery envelope.

> Don't confuse `trigger.*` (from `@secondlayer/sdk`, for **chain subscriptions**) with `on.*` (from `@secondlayer/stacks`, for **subgraph sources** in a handler config) ‚Äî they look similar but configure different things.

## Verify webhooks

Subscription deliveries are signed. Verify the signature before trusting a payload.

```ts
import { verifyWebhookSignature } from "@secondlayer/sdk";

const valid = verifyWebhookSignature(req.headers, rawBody, secret);
if (!valid) return new Response("bad signature", { status: 401 });
```

## Checkpointed consumer

For indexers and ETL jobs, `consume` polls and commits a cursor so you never miss or double-process events ‚Äî return the cursor you committed inside `onBatch`.

```ts
await sl.streams.events.consume({
  types: ["ft_transfer"],
  onBatch: async (events) => {
    // write your rows here, then return the last committed cursor
    return events.at(-1)?.cursor;
  },
});
```
