# Streams

Raw chain event firehose ŌĆö cursor-paginated REST, idempotent, replayable. The lowest level of indexing on Secondlayer: the inputs, not our decoding, for building an indexer from zero.

This is the exact surface [Index](/docs/index) is built on ŌĆö our decoder is itself a Streams consumer. Page forward with a cursor, replay from any point, and ŌĆö with a committed checkpoint, within your tier's retention ŌĆö never miss an event. Stream live over SSE in-process (see below), or fan out to your backend with [Subscriptions](/docs/subscriptions).

Streams reads require an API key ŌĆö send it as `Authorization: Bearer`. Free keys read the recent 24-hour window (free retention is 1 day); reaching older history is [pay-as-you-go credits](/docs/authentication#pay-as-you-go-credits) or a paid plan. A read below the free window returns `402 UPGRADE_REQUIRED`.

## Read the stream

```bash
curl -H "Authorization: Bearer $SL_API_KEY" \
  "https://api.secondlayer.tools/v1/streams/events?limit=100"
```

Each response returns a `next_cursor`. Pass it back as `cursor` to continue exactly where you left off ŌĆö deliveries are idempotent, so retries never double-count.

```bash
curl -H "Authorization: Bearer $SL_API_KEY" \
  "https://api.secondlayer.tools/v1/streams/events?cursor=7970329:8"
```

## Filter the stream

Narrow the firehose by event type, contract, or principal ŌĆö `types`, `contract_id`, `sender`, `recipient`, `asset_identifier`, and `from_height`/`to_height`. Filters are exact-match; event types without a given field simply don't match.

```bash
curl -H "Authorization: Bearer $SL_API_KEY" \
  "https://api.secondlayer.tools/v1/streams/events?types=ft_transfer&sender=SP...&limit=50"
```

## Build an indexer on it

The SDK ships the loop: `events.consume` is a checkpointed consumer with automatic reorg rewind. Write your rows inside `onBatch`, return the cursor you committed ŌĆö crash anywhere and the next run resumes from your checkpoint. Reorgs rewind the cursor to the fork point and re-deliver (at-least-once; your writes should be idempotent).

```ts
await streams.events.consume({
	types: ["ft_transfer"],
	fromCursor: await db.loadCheckpoint(),
	onBatch: async (events, envelope) => {
		await db.insertEvents(events);
		return envelope.next_cursor; // the checkpoint you committed
	},
	onReorg: async (reorg) => {
		await db.rollbackAbove(reorg.fork_point_height);
	},
});
```

For cold history, `events.replay` backfills from the signed parquet dumps, then tails live from the manifest's `latest_finalized_cursor` ŌĆö no gap or dupe at the seam:

```ts
await streams.events.replay({
	from: "genesis",
	async onDumpFile(file) {
		const bytes = await streams.dumps.download(file); // sha256-verified
		await ingestParquet(bytes);
	},
	async onBatch(events, envelope) {
		await db.insertEvents(events);
		return envelope.next_cursor;
	},
});
```

`from: "genesis"` replays every window the dumps cover ŌĆö the manifest's `coverage` reports that range, which starts where the dump program began (not necessarily chain block 1). To start a partial backfill, pass a cursor instead. Need decoded history all the way back? Use [Index](/docs/index), which serves the full decoded chain from `from_height=0`.

### Dumps + DuckDB

The dumps are plain parquet ŌĆö pull a verified range and query it locally, no indexer required:

```bash

duckdb -c "SELECT event_type, count(*) AS events
           FROM read_parquet('./dump/**/*.parquet')
           GROUP BY 1 ORDER BY 2 DESC;"
```

## Real-time firehose (SSE)

`GET /v1/streams/events/stream` streams events as Server-Sent Events over a single long-lived connection ŌĆö the native push surface inside Streams. Authenticate with `Authorization: Bearer`, and apply the same filters as `/events`: `types`, `not_types`, `contract_id`, `sender`, `recipient`, and `asset_identifier`.

```bash
curl -N -H "Authorization: Bearer $SL_API_KEY" \
  "https://api.secondlayer.tools/v1/streams/events/stream?types=ft_transfer"
```

Choose a start position with `cursor` (or `from_cursor`), or `from_height`. With no start position the connection live-tails from the reorg-clamped tip. A `ping` keepalive frame is sent every 20s so idle connections stay open.

The SDK's `client.events.subscribe(...)` is the ergonomic way to consume this ŌĆö it carries your Bearer key, auto-reconnects from the last delivered cursor, and verifies per-frame signatures. See the [SDK docs](/docs/sdk).

### Per-frame signatures

Each SSE frame body is `{ event, sig, key_id }`, where `sig` is an ed25519 signature over `JSON.stringify(event)`. When the Streams client is created with `verify`, every frame is checked against the key published at `/public/streams/signing-key` before your handler runs ŌĆö a bad or missing signature throws. This is the streaming counterpart to the signed `X-Signature` on REST reads: trust the data without trusting the server.

## What's new (May 2026)

- **Finality** ŌĆö `tip.finalized_height` plus a `finalized` flag on every event, anchored to Bitcoin (burn-block) confirmations. A `finalized` event won't reorg.
- **Caching** ŌĆö fully-finalized pages are served `Cache-Control: public, immutable` with an `ETag` (and `304` on `If-None-Match`); the moving tip stays short-lived.
- **Signed responses** ŌĆö every read carries an ed25519 `X-Signature`. Fetch the public key at `/public/streams/signing-key` and verify, or set `verify` on the SDK client ŌĆö trust the data without trusting the server.
- **Bulk + replay** ŌĆö download finalized history as parquet (`sl streams pull`, or the SDK `dumps` namespace) and `events.replay({ from: "genesis" })` to backfill cold history then tail live with no gap at the seam. The bulk manifest is itself ed25519-signed and is verified before any per-file `sha256` is trusted ŌĆö the manifest signature is the root of trust, with the file hashes chained beneath it. The client option `verifyDumpsManifest` defaults to `true` (both `dumps.list()` and `events.replay()` enforce it); opt out with `verifyDumpsManifest: false`.
