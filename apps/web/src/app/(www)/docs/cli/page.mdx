# CLI

The `sl` command-line tool — authenticate once, manage subgraphs and subscriptions from your terminal.

## Install

```bash
bun add -g @secondlayer/cli
```

## Common commands

```bash
sl login                 # authenticate (opens the browser)
sl subgraphs deploy      # deploy a subgraph from a config
                         #   --visibility public|private (managed → public, BYO → private)
sl subgraphs publish     # claim the global public name (409 PUBLIC_NAME_TAKEN if claimed)
sl subgraphs unpublish   # make a subgraph private again
sl subgraphs ls          # list your deployments
sl subscriptions create  # wire up a webhook subscription
```

Every tool shares one session — sign in once and the SDK, CLI, and MCP server all inherit it.

## Orientation

```bash
sl context               # headless orientation snapshot
```

`sl context` prints a snapshot to orient before acting — your account, the live Streams and Index tips, your subgraphs, your subscriptions, and any in-flight reindex operations. It's the CLI counterpart to the MCP `secondlayer://context` resource agents read before they reach for a tool.

`sl subscriptions create` wires a webhook to a subgraph table. [Direct chain subscriptions](/docs/subscriptions#direct-chain-subscriptions-no-subgraph) — webhooks on raw chain events with no subgraph — are created over the SDK, REST, or MCP instead (they take a `triggers` array rather than a table).

## ORM codegen (frozen)

`sl subgraphs codegen --target prisma|drizzle|kysely` generates a typed ORM
schema for a subgraph's tables, for BYO-database setups. Works, frozen — see
[Subgraphs](/docs/subgraphs#use-your-own-database--orm) if you need it.

## Local devnet (frozen)

`sl devnet` / `sl local` run the pipeline against a local Clarinet devnet. They
work but are frozen — no further investment; the supported dev loop is hosted
(`sl subgraphs create` → `deploy` against a recent block). Run `sl devnet --help`
if you need them.
