# CLI

The `sl` command-line tool: authenticate once, manage subgraphs and subscriptions from your terminal.

## Install

```bash
bun add -g @secondlayer/cli
```

## Common commands

```bash
sl login                 # authenticate (opens the browser)
sl subgraphs create      # scaffold a subgraph
                         #   --from-contract <id> infers typed print payloads (no login needed)
sl subgraphs deploy      # deploy a subgraph from a config
                         #   --visibility public|private (managed → public, BYO → private)
sl subgraphs publish     # claim the global public name (409 PUBLIC_NAME_TAKEN if claimed)
sl subgraphs unpublish   # make a subgraph private again
sl subgraphs ls          # list your deployments
sl subscriptions create  # wire up a webhook subscription
```

Every tool shares one session: sign in once and the SDK, CLI, and MCP server all inherit it.

## Orientation

```bash
sl context               # headless orientation snapshot
```

`sl context` prints a snapshot to orient before acting. It's the CLI counterpart to the MCP `secondlayer://context` resource agents read before they reach for a tool.

The snapshot covers:

- Your account
- The live Streams and Index tips
- Your subgraphs and subscriptions
- Any in-flight reindex operations

`sl subscriptions create` wires a webhook to a subgraph table.

<Callout type="note" title="Chain subscriptions use a different surface">

[Direct chain subscriptions](/docs/subscriptions#direct-chain-subscriptions-no-subgraph) (webhooks on raw chain events with no subgraph) are created over the SDK, REST, or MCP instead. They take a `triggers` array rather than a table.

</Callout>

## Scaffold from a contract

```bash
sl subgraphs create sbtc-registry \
  --from-contract SM3VDXK3WZZSA84XXFKAFAF15NNZX32CTSG82JFQ4.sbtc-registry
```

`--from-contract` infers each print topic's real payload shape from indexed
history (the [print-schema endpoint](/docs/index#print-schemas)) and writes a
typed definition. Anonymous, no login needed.

The generated definition includes:

- One `print_event` source per topic with a `prints` field map (fully typed `event.data`)
- A single wide table with per-column "null except on topics: …" comments
- Pass `--table-per-topic` for a normalized table per topic instead

`sl subgraphs codegen <file> --payloads` emits a `.d.ts` of per-topic payload
types (a discriminated union per source) next to the definition.

<Callout type="note" title="Payload codegen is print-only">

`--payloads` is incompatible with `--target`/`--models-only`/`--env` (those are ORM-only).

</Callout>

<Callout type="note" title="Deploy-time warnings">

`sl subgraphs deploy` also prints advisory `warnings` when a handler reads an
`event.data` field never observed on-chain for its source's topic. Warn-only,
never blocks.

</Callout>

## Generate typed contract clients

Turn Clarity contracts into typed TypeScript — local `.clar` files, deployed
contracts, or a whole Clarinet project.

```bash
sl contracts generate ./contracts/token.clar -o ./src/generated.ts
sl contracts generate SP3K8BC0PPEVCV7NZ6QSRWPQ2JE9E5B6N3PA0KBR9.alex-vault -o ./src/generated.ts
sl contracts generate --watch    # regenerate on .clar / config / Clarinet.toml changes
```

Deployed contract ids fetch the real ABI from the network (address prefix picks
mainnet vs testnet). Config-driven projects scaffold with `sl init` and
regenerate with plain `sl contracts generate`:

```ts
// secondlayer.config.ts
import { defineConfig } from "@secondlayer/cli";
import { clarinet, react } from "@secondlayer/cli/plugins";

export default defineConfig({
	out: "./src/generated/contracts.ts",
	plugins: [clarinet(), react()],
});
```

| Plugin | What it adds |
|---|---|
| `clarinet()` | Real ABIs from your Clarinet project via Simnet — includes `[project.requirements]` dependency contracts (`includeRequirements: false` to opt out) |
| `react()` | Typed React hooks per contract |
| `testing()` | Clarinet SDK test helpers |

Generated output ships named type aliases per function (`TokenTransferArgs`,
`TokenTransferResult`) and an ABI const branded with `TypedAbi`. Feed it to
`getContract` from [`@secondlayer/stacks`](/docs/stacks#also-in-the-box) and
hovers and type errors show the named aliases — not a wall of expanded
conditional types.

<Callout type="tip" title="Your contract's dependencies get types too">

Contracts you depend on but didn't write — a token you call, a protocol you
integrate — are declared as Clarinet requirements. The `clarinet()` plugin
generates typed clients for those by default.

</Callout>

## Frozen features

These work but get no further investment. The supported dev loop is hosted (`sl subgraphs create` → `deploy` against a recent block).

- `sl subgraphs codegen --target prisma|drizzle|kysely` generates a typed ORM schema for a subgraph's tables (BYO-database). See [Subgraphs](/docs/subgraphs#use-your-own-database--orm).
- `sl devnet` / `sl local` run the pipeline against a local Clarinet devnet. Run `sl devnet --help` for usage.
