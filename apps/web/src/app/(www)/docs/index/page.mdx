# Index

The decoded read layer for Stacks ŌĆö we run the indexer, you query. One REST surface over decoded events, contract calls, the canonical block map, blocks, full transaction documents, PoX-4 stacking, and the pending mempool. No decoders to write, no node to run. Open beta: reads need no key (free-tier keys are rejected; Build+ for keyed access).

Every list response uses the same cursor envelope ŌĆö the collection, a `next_cursor`, the chain `tip`, and `reorgs` where applicable. Cursors are opaque `<block_height>:<index>` resume tokens; pass them back unchanged to continue.

## Surfaces

| Endpoint | Returns |
| --- | --- |
| `/v1/index/events` | Decoded events for a chosen `event_type` (+ `ft-transfers` / `nft-transfers` aliases) |
| `/v1/index/contract-calls` | Decoded contract-call transactions with `args` and `result` |
| `/v1/index/transactions` ┬Ę `/{tx_id}` | Full transaction documents (fee, nonce, post-conditions, decoded payload) |
| `/v1/index/blocks` ┬Ę `/{height_or_hash}` | Decoded blocks, list or single |
| `/v1/index/canonical` | Canonical block-hash map ŌĆö one row per height, for chain sync |
| `/v1/index/stacking` | Decoded PoX-4 stacking actions (stack-stx, delegate-stx, ŌĆ”) |
| `/v1/index/mempool` ┬Ę `/{tx_id}` | Pending (unconfirmed) transaction documents |
| `/v1/index/usage` | Your own consumption and tier limits |

## Events

```bash
curl "https://api.secondlayer.tools/v1/index/events?event_type=stx_transfer&limit=20"
```

Allowed filters vary by `event_type`: `stx_*` have no `contract_id`, `nft_*` accept `asset_identifier`, and `print` filters by `contract_id` and carries a `payload` with the Clarity value decoded to JSON. `ft-transfers` and `nft-transfers` are typed aliases for the two most common types.

## Contract calls

```bash
curl "https://api.secondlayer.tools/v1/index/contract-calls?function_name=transfer&limit=20"
```

Each row decodes the call's `args` and `result` from the raw Clarity. Contract-call cursors are a separate keyspace from event cursors.

## Transactions

```bash
curl "https://api.secondlayer.tools/v1/index/transactions/0x<tx_id>"
```

Full transaction documents, decoded from the raw transaction: `fee`, `nonce`, `sponsored`, `post_conditions`, and a typed `payload` sub-object. List with `/v1/index/transactions` filtered by `type`, `sender`, or `contract_id`, or fetch one by `tx_id`.

### Proofs

```bash
curl "https://api.secondlayer.tools/v1/index/transactions/0x<tx_id>/proof"
```

`/{tx_id}/proof` returns a trustless inclusion proof ŌĆö the raw transaction, raw header, and merkle path needed to verify, client-side, that the transaction is in a Stacks block and that the reward cycle's signers attested to it. Nothing Secondlayer returns is trusted. See [Verification](/docs/verification).

## Blocks and canonical

```bash
curl "https://api.secondlayer.tools/v1/index/blocks/892100"
```

`/v1/index/blocks` lists decoded blocks (or fetch one by height or hash). `/v1/index/canonical` returns the canonical block-hash map ŌĆö exactly one row per height, orphaned blocks excluded ŌĆö so a client can sync only the canonical chain.

## Stacking

```bash
curl "https://api.secondlayer.tools/v1/index/stacking?function_name=stack-stx&limit=20"
```

Decoded PoX-4 stacking actions, filterable by `function_name`, `stacker`, or `caller`.

## Mempool

```bash
curl "https://api.secondlayer.tools/v1/index/mempool?contract_id=SP000000000000000000002Q6VF78.pox-4"
```

Pending, unconfirmed transactions with the same decoded enrichment as `/transactions`, but pre-chain: no `block_height` or `result`, plus a `received_at`. Filter by `sender`, `type`, or `contract_id`. Rows leave the mempool on confirmation or drop, so the single form 404s once a transaction mines. This surface is never cached.

## Discovery

`GET /v1/index` describes itself. Alongside the route list it returns `event_type_filters` ŌĆö a per-type map of `columns`, `allowed_filters`, `equality_filters`, and `required_non_null`, generated from the event registry ŌĆö so an agent can learn exactly what each `event_type` accepts at runtime rather than hard-coding it.

## Reorgs

Read envelopes (e.g. `/v1/index/stacking`, `/transactions`, `/contract-calls`) carry a `reorgs` array alongside the data ŌĆö the chain reorganizations overlapping the page's height range, empty when none. Use it to reconcile any rows you've already committed from a fork.

```json
{
  "reorgs": [
    {
      "id": "...",
      "detected_at": "2026-06-05T12:00:00Z",
      "fork_point_height": 892100,
      "old_index_block_hash": "0xŌĆ”",
      "new_index_block_hash": "0xŌĆ”",
      "orphaned_range": { "from": "892101:0", "to": "892140:7" },
      "new_canonical_tip": "892140:3"
    }
  ]
}
```

To reconcile: roll back any rows whose cursor falls inside `orphaned_range`, then re-fetch from `new_canonical_tip`. Both bounds are `<block_height>:<event_index>` cursors (reorg cursors track decoded-event positions; on transaction-keyed feeds like `/transactions` and `/contract-calls`, compare by the block-height component). The SDK exports the `IndexReorg` type from `@secondlayer/sdk` for typing these elements.

```ts
	// ŌĆ”

```

## Usage

Read your own consumption and tier limits at any time.

```bash
curl "https://api.secondlayer.tools/v1/index/usage" -H "Authorization: Bearer $SL_API_KEY"
# ŌåÆ { product, tier, limits: { rate_limit_per_second }, usage: { decoded_events_today, decoded_events_this_month } }
```

Also `sl.index.usage()` (SDK) and `index_usage` (MCP). See the [REST API](/docs/rest-api) reference for the full per-type filter grammar and envelope shapes.
