# Sprint Plan — Surface migration plan on BYO breaking-change refuse path

## Diagnosis

When a BYO subgraph deploy hits a breaking schema change (removed tables/columns, changed column types) or a forced reindex, `deploySchema` calls `refuseDestructiveOnByo(reason)` → bare `throw new Error(...)` (`packages/subgraphs/src/schema/deployer.ts:197-203`, called at `:276` and `:296-300`). No code, no structured payload.

- **API** (`packages/api/src/routes/subgraphs.ts:413-423`): no try/catch → bare Error lacks `code` → global middleware (`packages/api/src/middleware/error.ts`) falls to **500** `{error:"Internal Server Error",code:"INTERNAL_ERROR"}`. Reason string swallowed.
- **SDK** (`base.ts:139-144`): `status>=500` short-circuit → generic "Server error". Message lost.
- **CLI** (`commands/subgraphs.ts:875-878`): catch-all `error("Failed to deploy subgraph: " + err)`. No diff/plan/DDL.

**Gap.** Helper machinery exists — `diffSchema()`, `hasBreakingChanges()` (reasons), `renderDeployPlan()` (statements+grantScript), `DeployDiff`/`DeployPlan` — but discarded on refuse. Make refuse path **additive**: same refusal, but carry `{reasons,diff,plan:{statements,dropStatement,grantScript}}` end-to-end → **422** → copy/paste DDL.

**Verified facts (corrections vs draft):**
- `refuseDestructiveOnByo` current signature = `(reason: string): never` — introduce a new signature. The TWO call sites differ: `:276` (same-hash force reindex) has **no `diff` (TableDiff) in scope**; `:296-300` has `diff` + `reasons` in scope. Helper must accept **optional** diff. `:308-317` (`deployDiff` mapping) only runs on the **non-refuse** branch → extract a `toDeployDiff(diff, reasons)` helper, reuse at both the refuse-call and the existing non-refuse mapping.
- `CODE_TO_STATUS` type is `Record<string, 400|401|403|404|409|423|429>` — **no 402**. But `error.ts:48-51` re-narrows to `400|401|402|403|404|409|423|429` (**includes 402**) — drift. Add `422` to BOTH; optionally drop the phantom 402 from error.ts to realign (call out, don't block on it).
- `renderDeployPlan` (`:135-150`) emits **no** drop. `DeployPlan` (`:123-129`) = `{schemaName,statements,grantScript}`.
- Managed reindex DROP DDL (`:278`,`:302`) = `DROP SCHEMA IF EXISTS "<schema>" CASCADE` — match this exactly (keep a trailing `;` for the user-facing copy-paste).
- Dry-run wire shape is **field-selected** in the route (`:373-379` returns `grantScript`; `:385-390` does NOT) → adding `dropStatement` to `DeployPlan` does **not** change dry-run JSON. **Q3 resolved: always include `dropStatement`; no wire change.**
- SDK `errors.ts` index export at `packages/sdk/src/index.ts:101`. SDK `base.ts` already parses body+code into `ApiError` (status<500); 422 reaches `:146-165`. `ApiError.body` preserves parsed JSON → plumbing works; typed subclass = ergonomics only.
- Local CLI deploy (`commands/subgraphs.ts:846-849`) does **NOT** pass `dataDb`/`databaseUrlEnc` → **local BYO breaking refusal is unreachable today** (Q2 real; scope T7 to defensive duck-typed guard).
- Both remote+local deploy share the same `catch` (`:875-878`).
- Existing tests to extend: `packages/subgraphs/test/deploy-byo.test.ts`, `packages/api/test/subgraphs-deploy.test.ts`, `packages/sdk/src/__tests__/base.test.ts`.

**Decisions baked in:**
1. `renderDeployPlan` → `DeployPlan` gains `dropStatement: string` (always populated; harmless).
2. New code `BYO_BREAKING_CHANGE` → **422**. Confirm 422 over 409 (Q5).
3. SDK adds typed `ByoBreakingChangeError extends ApiError` (ergonomics; plumbing already works via `ApiError.body`).
4. Detail interface single-sourced in `@secondlayer/shared` (`ByoBreakingChangeDetails`), reused by subgraphs + sdk (resolves Q6 toward consolidation).
5. **No destructive `--force`** this sprint — copy points to manual DROP+rebuild (Q1).

**Scope guard re SQL-injection/numeric checklist:** this item is error-plumbing only — adds NO new SQL execution, NO aggregates, NO column allowlist surface. `dropStatement`/`statements` are rendered strings from `generateSubgraphSQL`/`pgSchemaName` (existing identifier quoting) shown to the user, never auto-executed on the BYO path. No new injection/aggregate/numeric surface. (Pre-existing generator quoting unchanged — out of scope.)

**Release order (consumers built from dist):** `shared`+`subgraphs` first → then `api`(private)+`sdk`+`cli`.

## Chosen shapes

### `@secondlayer/shared` — single-source detail interface
```ts
// packages/shared/src/errors.ts (or a schemas module if barrel prefers)
export interface ByoBreakingChangeDetails {
  reasons: string[];
  diff: { addedTables: string[]; removedTables: string[]; addedColumns: Record<string,string[]>; breakingChanges: string[] };
  plan: { schemaName: string; dropStatement: string; statements: string[]; grantScript: string };
}
```

### `@secondlayer/subgraphs`
```ts
export interface ByoMigrationPlan { schemaName: string; dropStatement: string; statements: string[]; grantScript: string }
export class ByoBreakingChangeError extends Error {
  readonly code = "BYO_BREAKING_CHANGE" as const;
  readonly details: ByoBreakingChangeDetails;
  constructor(reasons, diff, plan) { super("Breaking schema change on a BYO subgraph would drop data in your database. Review the plan and run the DROP + rebuild DDL manually."); this.name="ByoBreakingChangeError"; this.code="BYO_BREAKING_CHANGE"; this.details={reasons,diff,plan}; }
}
```
- Plain `Error` (NOT `SecondLayerError`) with literal `code` field → matched by `error.ts:45-65` code-based path across bundle boundary (avoids bunup cross-bundle `instanceof`).

### `renderDeployPlan` extension
`DeployPlan` += `dropStatement: string`; signature unchanged. `dropStatement = \`DROP SCHEMA IF EXISTS "${schema}" CASCADE;\``.

### API 422 body
