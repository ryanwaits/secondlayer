## Sprint A — Shared leader util + scale-safe in-place (no new service yet)

**Goal:** evaluator + chain-reorg + catch-up safe to run on N processors (leader-gated), still inside subgraph-processor. Demoable: boot 2 subgraph-processor replicas locally → only one evaluates/reorg-rewinds/catches-up; delivery unaffected. **Pre-req for Sprint B's two-deploy cutover** (multiple evaluators must be safe before a second service boots one).

> **A-invariant (B2 pre-req):** the evaluator's `advanceCursor` AND `handleChainReorg`'s rewind mutate the **same** `trigger_evaluator_state` row (verified: `chain-reorg.ts:118-126` rewinds `last_processed_block` to `forkHeight-1` on target; `trigger-evaluator-loop.ts:50-66` advances it). They MUST run under **one** lock (`SUBSCRIPTION_EVALUATOR_LOCK_KEY`), else a B2-window reorg-poll on one process races a cursor-advance on another. Only **two** subscription/subgraph lock keys total: evaluator(+chain-reorg) and catch-up. Emitter stays lock-free (competing-consumer via FOR UPDATE SKIP LOCKED).

> **Leader DB-home (verified):** `trigger_evaluator_state`, `subscription_outbox`, and the `subgraphs` table all live on **TARGET** (`chain-reorg.ts:5,46` `getTargetDb()`; P0 control-plane split). But `createPostgresLeaderBackend()` (leader.ts:41-42) resolves `leaderDatabaseUrl()` = `SOURCE_DATABASE_URL||DATABASE_URL` (leader.ts:32-38) → **SOURCE**. The lock MUST sit on the same DB as the serialized row, so the backend factory needs an explicit target URL (A1).

- [ ] **A1: Promote leader util to `@secondlayer/shared` + accept an explicit lock-DB URL.** Move `packages/indexer/src/leader.ts` → `packages/shared/src/leader.ts`; switch its `@secondlayer/shared/logger` import to relative; add `./leader` to shared `exports`. Change `createPostgresLeaderBackend(url?: string)` to accept an explicit URL (default = existing `leaderDatabaseUrl()` so indexer behavior is unchanged). Move `leader.test.ts` → shared (re-point imports). In indexer barrel (`index.ts:51-53`) re-export `withLeaderLock`/`INDEXER_LEADER_LOCK_KEY` from `@secondlayer/shared`; delete the indexer copy. Add lock-key constants `SUBSCRIPTION_EVALUATOR_LOCK_KEY`, `SUBGRAPH_CATCHUP_LOCK_KEY` (distinct ints; no separate chain-reorg key). → **validates:** `bun run build` + `bunx tsc -b`; relocated leader test green; changesets `shared` (minor) + indexer.
- [ ] **A2: Leader-gate the trigger evaluator (whole loop, not per-tick).** Pass the existing `startTriggerEvaluator` (its own `setTimeout` loop, `trigger-evaluator-loop.ts:114-141`) as the `startWork` to `withLeaderLock(SUBSCRIPTION_EVALUATOR_LOCK_KEY, startTriggerEvaluator, {createBackend:()=>createPostgresLeaderBackend(targetLeaderUrl())})` — matches the indexer pattern (`index.ts` singleton loops). Non-leaders run no loop at all; on relinquish the loop stops. `targetLeaderUrl()` = `TARGET_DATABASE_URL||targetListenerUrl()`. → **validates:** new `trigger-evaluator-loop.test.ts`: two instances sharing one test DB → only the lock holder advances `last_processed_block`; on holder stop the other acquires within `pollMs`. changeset subgraphs.
- [ ] **A3: Co-gate chain-reorg under the evaluator lock.** Make `handleChainReorg`'s invocation run only on the evaluator leader (same `SUBSCRIPTION_EVALUATOR_LOCK_KEY` session). Parameterize `startStreamsReorgPoll` to split its two handlers: `onChainReorg` fires only when this process holds the evaluator lock; `onSubgraphReorg` (`handleSubgraphReorg`, idempotent row-deletes) stays on the subgraph plane unchanged. → **validates:** unit test: a process that is NOT evaluator-leader invokes `handleChainReorg` zero times but still runs `handleSubgraphReorg`; build. changeset subgraphs.
- [ ] **A4: Leader-gate subgraph catch-up.** Factor `processor.ts:437/452/485` call sites into one `runCatchUp()` helper, gate the whole driver under `withLeaderLock(SUBGRAPH_CATCHUP_LOCK_KEY, …, {createBackend:()=>createPostgresLeaderBackend(targetLeaderUrl())})`. Keep the process-local `catchingUp` Set (`catchup.ts:22`) as the in-process guard; leader lock is the cross-process guard. → **validates:** test: two processors, NOTIFY both → only leader runs `catchUpAll` (spy/row-count); idempotent upserts keep correctness either way. changeset subgraphs.

## Sprint B — Extract subscription-processor (two-deploy cutover)

**Goal:** dedicated `subscription-processor` service (emitter + evaluator + chain-reorg) deployable/scalable apart from subgraph indexing. Demoable after Deploy-2: kill subgraph-processor → webhooks still deliver; kill subscription-processor → subgraph indexing still runs. **Strict two-deploy ordering** to avoid a no-emitter window. **Depends on Sprint A** (multiple evaluators must be leader-safe).

- [ ] **B1: New `subscription-processor` entrypoint + shared plane helper.** Factor the subscription concerns out of `startSubgraphProcessor` into `startSubscriptionPlane()` (emitter + evaluator + chain-reorg poll). Add `packages/subgraphs/src/subscription-service.ts` (`SERVICE_NAME="subscription-processor"`, heartbeat mirroring `service.ts:34`) booting only `startSubscriptionPlane()`. `startSubgraphProcessor` still calls `startSubscriptionPlane()` for now (removed in B3). → **validates:** `bun run packages/subgraphs/src/subscription-service.ts` boots + heartbeats locally against test DB; build. changeset subgraphs.

## Sprint D — Authenticity (universal sig + signed bulk manifest)

**Goal:** every webhook + the cold bulk lane carry verifiable Secondlayer authenticity. D1 independent; D2 has strict signing-then-verify rollout.

- [ ] **D1: universal webhook signature header.** **Resolve Q3 first (blocks D1's shape).** In `formats/index.ts` `buildForFormat` attach `webhook-id` + a new `X-Secondlayer-Signature` to ALL 6 formats regardless of body shape — keep each format's body untouched. Export a **new, distinct** `verifySecondlayerSignature` from `@secondlayer/shared` and SDK (do **not** re-point the existing `verifyWebhookSignature`, which is hardwired to standard-webhooks HMAC headers `webhook-id/-timestamp/-signature` — `sdk/webhooks.ts:120` — re-pointing it breaks current verifiers). Note: HMAC needs a per-sub secret at verify time (none exists for `raw`/`cloudevents`), so ed25519 (single platform key) is the natural fit — confirm via Q3. → **validates:** `formats.test.ts` asserts every format carries a verifiable sig header; round-trip sign→verify; existing standard-webhooks verify still green; build. changesets shared + subgraphs + sdk.
- [ ] **D2a: sign the bulk manifest at export.** Add a `./crypto/ed25519` entry to shared `exports` (currently only `./crypto/standard-webhooks` + `./crypto/secrets` are exported — verified), OR import `{ ed25519 }` from the shared **root barrel** like the SDK does (`client.ts:1`). Add `signature` + `key_id` to `StreamsBulkManifest` (`manifest.ts:7-34`); in `exporter.ts:121` sign the canonical manifest JSON with `STREAMS_SIGNING_PRIVATE_KEY`. Publish signed `latest.json` + history; **backfill** existing manifests with a one-shot re-sign script. → **validates:** export test asserts manifest carries valid ed25519 sig verifiable with the published key; build. changeset shared (exports change) + indexer.
- [ ] **D2b: SDK verifies manifest sig (default OFF).** `dumps.ts list()` verifies `manifest.signature` against the published key BEFORE trusting any file sha256; behind opt-in flag (default off) until D2a backfill lands. Flip default ON in a follow-up release after backfill confirmed. → **validates:** test: tampered manifest fails verify when flag on, passes-through when off; build. changeset sdk. **Order: D2a + backfill MUST land before D2b default-on.**

## Sprint E — Streams SSE lane

**Goal:** "Streams" gets a true push surface at chain cadence (not poll-bounded), reusing the immutable envelope + ed25519 signing. Demoable: `client.events.subscribe()` receives new canonical events live.

- [ ] **E1: `GET /v1/streams/events/stream` SSE endpoint.** **Resolve Q4 first (blocks E1's frame shape).** Add to `routes/streams.ts` a `streamSSE` handler (mirror `subgraphs.ts:1341-1408`): seek from a `<height>:<event_index>` cursor, poll `readCanonicalStreamsEvents…` for new canonical events past cursor, 20s `ping` heartbeat, exit on `stream.aborted`, respect retention/tip-stable bounds. **SSE frames carry no per-frame headers**, so the existing header-based `respondSignedJson`/`X-Signature` contract (`signing.ts:51-60`) does NOT apply — define a new **inline-sig envelope** `{ event, sig, key_id }` signed per frame via `getStreamsSigner().sign(JSON.stringify(event))`. → **validates:** integration test: inline-signed event frames + heartbeat, sig verifies against published key; build. changeset api.
- [ ] **E2: SDK `client.events.subscribe()` + frame verifier.** Add to the SDK streams client an `EventSource`-based `subscribe(onEvent, {cursor, signal})` mirroring `subgraphs/client.ts:323-365`: parse JSON frames, ignore `ping`. Add a **new frame-verify routine** for the inline-sig envelope (the existing streams verify reads response *headers*, `client.ts:69` — it can't verify embedded-field frames, so this is new code, not reuse). Expose cursor for resume, Node<22 guard. → **validates:** SDK test with mock EventSource: `onEvent` called on good frames, `ping` ignored, bad-sig frame rejected; build. changeset sdk.
