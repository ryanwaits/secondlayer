## Design decisions

### Request
`GET /api/subgraphs/:subgraphName/:tableName/aggregate`

Reuse count/list filter surface (`status=active`, `price.gte=2000000`, `col.in=a,b`, `_search`). Aggregates via control params:
- `_count=true` → `COUNT(*)`
- `_countDistinct=col1,col2` → `COUNT(DISTINCT col)` per col
- `_sum=col1,col2` / `_min=col1,col2` / `_max=col1,col2`

Rules:
- No agg param → default `{ count }` only.
- `_sum/_min/_max` cols MUST be `tableDef.columns[col].type ∈ {uint,int}` (system `_block_height` numeric → allowed) → else `400 NON_NUMERIC_COLUMN`.
- `_countDistinct` any allowlisted col.
- All agg cols validated vs `getValidColumns(tableDef)` → `400 INVALID_COLUMN` (reuse `InvalidColumnError`).
- Strip `_count/_countDistinct/_sum/_min/_max` BEFORE `parseQueryParams` (else l.211 throws).
- Per-request cap: ≤ 32 agg columns total → `400 TOO_MANY_AGGREGATES` (cost guard).

### Response
```json
{
  "count": 42,
  "countDistinct": { "seller": 7 },
  "sum":  { "price": "12300000000" },
  "min":  { "price": "1000000", "_block_height": "840000" },
  "max":  { "price": "9900000000", "_block_height": "851234" }
}
```
- Keys present only for requested aggs.
- `count`, `countDistinct.col` → JSON `number` (`Number.parseInt`; counts << 2^53).
- `sum/min/max` → **string** (lossless). `sum` of empty set → `"0"` (`COALESCE(SUM,0)::text`, matches ctx). `min/max` → `string | null` (null when empty/all-null).

### SQL (single round-trip, parameterized)
```sql
SELECT
  COUNT(*)                          AS count,
  COUNT(DISTINCT "seller")          AS "cd__seller",
  COALESCE(SUM("price"),0)::text    AS "sum__price",
  MIN("price")::text                AS "min__price",
  MAX("price")::text                AS "max__price"
FROM "subgraph_x"."listings"
WHERE "status" = $1
```
- Every col `ident()`-quoted. Aliases `kind__col` (col already passed `getValidColumns` ∩ `/^[a-z0-9_]+$/` so alias is ident-safe).
- WHERE via existing `buildWhereConditions(parsed, params)` (incl. `_search`).

### Shared types — `packages/shared/src/schemas/subgraphs.ts`
```ts
export interface SubgraphAggregateParams {
  filters?: Record<string, string>;
  count?: boolean;
  countDistinct?: string[];
  sum?: string[];
  min?: string[];
  max?: string[];
}
export interface SubgraphAggregateResponse {
  count?: number;
  countDistinct?: Record<string, number>;
  sum?: Record<string, string>;
  min?: Record<string, string | null>;
  max?: Record<string, string | null>;
}
```

### SDK typed method + inference — `packages/subgraphs/src/infer.ts`
Use `const` type param (TS ≥5.0) so callers DON'T need `as const`:
```ts
// numeric cols only (uint|int → bigint, incl. nullable bigint|null and _blockHeight)
type NumericKeys<TRow> = {
  [K in keyof TRow]-?: NonNullable<TRow[K]> extends bigint ? K : never;
}[keyof TRow] & string;
type AnyKey<TRow> = keyof TRow & string;

export interface AggregateSpec<TRow> {
  count?: boolean;
  countDistinct?: AnyKey<TRow>[];
  sum?: NumericKeys<TRow>[];
  min?: NumericKeys<TRow>[];
  max?: NumericKeys<TRow>[];
  where?: WhereInput<TRow> & SystemWhereAliases;
}
type ColsOf<A, K extends keyof A> =
  A[K] extends readonly (infer C extends string)[] ? C : never;

export type AggregateResult<TRow, A extends AggregateSpec<TRow>> =
  (A["count"] extends true ? { count: number } : unknown) &
  (A["countDistinct"] extends readonly string[] ? { countDistinct: Record<ColsOf<A,"countDistinct">, number> } : unknown) &
  (A["sum"] extends readonly string[] ? { sum: Record<ColsOf<A,"sum">, string> } : unknown) &
  (A["min"] extends readonly string[] ? { min: Record<ColsOf<A,"min">, string | null> } : unknown) &
  (A["max"] extends readonly string[] ? { max: Record<ColsOf<A,"max">, string | null> } : unknown);
```
Add to `SubgraphTableClient<TRow>` (l.120):
```ts
aggregate<const A extends AggregateSpec<TRow>>(spec: A): Promise<AggregateResult<TRow, A>>;
```
Notes: `NonNullable<TRow[K]> extends bigint` cleanly picks numeric incl. nullable; uses `unknown` (not `{}`) for false branch → biome-clean. `const A` removes `as const` burden; if TS const-param inference proves flaky in test, fall back to documenting `as const`.
