# FOCUS VERDICT — Secondlayer, 2026-06-10

## 1. One-sentence product definition

**Secondlayer is the hosted indexer for Stacks: curl decoded chain data keyless in 10 seconds, or deploy a one-file TypeScript indexer and get your own hosted Postgres tables behind a public REST API — no node, no infra.**

## 2. Target product shape

**Two products. Index (free keyless wedge) + Subgraphs (the monetizable core).** Everything else is a named feature of one of them:

- **Webhooks** (today's Subscriptions) — a tab/feature of Index + Subgraphs. Name "Subscriptions" stays per ruling; positioning becomes feature-level.
- **Dumps** (today's Streams external) — signed parquet + `sl streams pull`, marketed as a feature; the Foundation-grant/researcher artifact replacing Datasets.
- **Explore** — discovery/social proof for public subgraphs, only after it's seeded.
- **MCP server** — distribution channel, not a product noun.

**Mental-model sentence (ship verbatim on docs landing, README, SKILL.md, MCP description):** "Curl /v1 for decoded Stacks data right now; deploy a one-file subgraph for your own tables with the same public API; attach a webhook to either."

**Golden-path 60-second journey:** Homepage hero IS a copy-pasteable `curl 'https://api.secondlayer.tools/v1/index/events?event_type=ft_transfer&contract_id=<theirs>&limit=5'` → decoded JSON in <10s, keyless (live-verified, already works) → one printed next step: `bunx @secondlayer/cli subgraphs create my-token --template sip-010-balances` → edit one file → `sl login` + `sl subgraphs deploy` (defaults to recent-block start, NOT genesis) → deploy output prints the public curl URL for their new table + "attach a webhook" as line 3 → first rows in minutes, subgraph visible on Explore.

**Target concept count: 5** (decoded event + cursor envelope, subgraph file = source/schema/handler, deploy, public table URL, webhook). Today's marketed count is 20+. Nav: Index / Subgraphs / Explore / Pricing. One tagline ("Index the chain. Own your API." — page.tsx:77-84); kill "agent-native data plane" and "the data plane for Stacks" variants.
