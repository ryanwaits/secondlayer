§4

Risk Map

1
Failover is not yet a productized operation
The roadmap gate names hot-spare automated failover and two rehearsals. Current repo evidence shows node server setup, but no promotion script, switch criterion, or rehearsal log.
→ fix: define primary/spare topology, promotion command, rollback command, and two timed rehearsals.
2
Public status is freshness-first, not SLO-complete
Streams and Index freshness are visible. API p50/p95, error rate, node health, and recent reorg visibility are still absent from the public status contract.
→ fix: add in-process rolling metrics or durable request samples and render them publicly.
3
Metering evidence is incomplete for public product endpoints
Platform account requests are counted, but Streams events returned and Index decoded rows returned are not yet captured as first-class usage units.
→ fix: emit product usage rows after successful paid Streams/Index responses.
4
Pricing copy contradicts the locked product catalog
PRODUCTS.md locks Free, Build, Scale, and Enterprise. The public pricing page still presents Launch and older dedicated-hosting language.
→ fix: update the web pricing page to the catalog after PRD acceptance.
