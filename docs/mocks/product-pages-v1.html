Streams

The raw firehose — every event the chain emits, ordered, cursor-paginated, reorg-aware. Resume from any cursor, hold the tip over SSE, or replay history from signed parquet dumps. No key for reads.

Machine access
  • GET/v1/streams/events?cursor=…cursor-paginated reads⧉
  • GET/v1/streams/events/tailSSE at the tip⧉
  • GETdata.secondlayer.tools/…/manifest/latest.jsonsigned bulk dumps⧉

In three lines

// resume exactly where you stopped
for await (const batch of sl.streams.consume({ cursor })) {
  await handle(batch.events);
}

What you get

Ordered + reorg-aware

Events arrive in canonical order; reorgs surface as explicit frames, never silent rewrites.

Resume from any cursor

Cursors are stable checkpoints. Crash, redeploy, pick up exactly where you left off.

Signed bulk history

Parquet dumps with ed25519-signed manifests — verify before you trust a byte.

Home/Index

Index

Decoded chain data, queryable. Every FT and NFT transfer, contract call, and print event — typed, filterable by contract, principal, or trait. Anonymous reads, wildcard CORS.

Machine access
  • GET/v1/index/events?event_type=…decoded events⧉
  • GET/v1/index/ft-transfers?contract_id=…token transfers⧉
  • GET/v1/index/nft-transfersnft activity⧉

In three lines

// decoded sBTC transfers, typed
const { events } = await sl.index.ftTransfers({
  contractId: "SM3V…sbtc-token",
});

// every SIP-010 at once, by trait
await sl.index.events({ trait: "sip-010" });

What you get

Decoded, not raw hex

Clarity values arrive as typed JSON — principals, uints, tuples — no client-side ABI wrangling.

Query by trait

One filter spans every SIP-010 or SIP-009 contract — declared and inferred implementations.

One pagination idiom

Cursor in, next_cursor out, reorg-aware — the same shape on every read surface.

Home/Subgraphs

Subgraphs

Your own indexer, minus the node. Define sources, schema, and handlers in one TypeScript file — deploy and get typed Postgres tables, a public read API, and a page on Explore, synced from genesis.

Machine access
  • GET/v1/subgraphspublic directory, no key⧉
  • GET/v1/subgraphs/:name/:tablecursor-paginated rows⧉
  • GET/v1/subgraphs/:name/docs.mdper-subgraph docs for agents⧉

In three lines

export default defineSubgraph({
  sources: { transfer: { type: "ft_transfer" } },
  schema: { transfers: { columns: { … } } },
  handlers: { transfer: (e, ctx) => ctx.insert("transfers", e) },
});

What you get

Public by default

Managed deploys get a shareable /v1 URL and a listing on Explore — flip to private anytime.

Synced from genesis

Backfill runs on our infra against the full chain history. You deploy a file, not a fleet.

Typed end to end

Generated clients for SDK, Kysely, Drizzle — your schema is the single source.

Home/Subscriptions

Subscriptions

Webhooks when it happens. Subscribe to chain events or your subgraph rows and get signed deliveries with retries and circuit breakers — the delivery layer of Index and Subgraphs.

Machine access
  • POST/api/subscriptionscreate (key required)⧉
  • GET/api/subscriptions/:id/deliveriesdelivery log⧉
  • POST→ your webhooksigned payloads⧉

In three lines

await sl.subscriptions.create({
  name: "whale-alerts",
  triggers: [trigger.ftTransfer({
    assetIdentifier: "SM3V…::sbtc-token",
    minAmount: 100_000_000 })],
  url: "https://hooks.example.com/sbtc",
});

What you get

Signed deliveries

Every POST carries an HMAC signature — verify provenance before you act on it.

Retries + circuit breakers

Exponential backoff, dead-letter visibility, automatic pause on persistent failures.

Chain or subgraph triggers

Fire on raw chain events or on rows landing in your own tables.
