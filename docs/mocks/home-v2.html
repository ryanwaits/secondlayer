Consume the raw firehose

Every event the chain emits — ordered, cursor-paginated, reorg-aware. Resume from any cursor, replay history from signed parquet dumps, or hold an SSE connection at the tip. This is the layer you'd run a node for.

Read the docs → 
// resume the firehose from your cursor
for await (const block of sl.streams.consume({ cursor })) {
  await handle(block.events);  // ordered, reorg-aware
}

// or bulk history from signed parquet dumps
const dumps = await sl.streams.dumps.list();

Decoded events, no key required

Every FT and NFT transfer, contract call, and print event — decoded, typed, and cursor-paginated. Filter by contract, principal, or trait. Anonymous reads, wildcard CORS.

Your own indexer, minus the node

Define sources, schema, and handlers in one TypeScript file. Deploy it and get typed Postgres tables, a public read API, and a page on Explore — synced from genesis without running anything.

Read the docs → 
export default defineSubgraph({
  name: "sbtc-flows",
  sources: { transfer: { type: "ft_transfer",
    assetIdentifier: "SM3V…sbtc-token::sbtc" } },
  schema: {
    transfers: { columns: {
      sender:    { type: "principal", indexed: true },
      recipient: { type: "principal", indexed: true },
      amount:    { type: "uint" } } },
    balances: { columns: {
      address: { type: "principal", unique: true },
      balance: { type: "uint" } } },
  },
  handlers: { transfer: async (e, ctx) => { … } },
});

The same API, from your shell

Every surface is also a command. Scaffold, deploy, query, tail — JSON by default, pipeable, scriptable. Local devnet included.

Read the docs → 
# contract → subgraph in one line
sl subgraphs scaffold SM3V…sbtc-token -o sbtc.ts

# deploy + watch it sync
sl subgraphs deploy sbtc.ts
sl subgraphs status sbtc-flows

# query from the shell, pipe to jq
sl subgraphs query sbtc-flows transfers --limit 3

Get started

Run it yourself. Or hand it to your agent.

Reads need no key, deploys need one command — and the whole surface speaks MCP, so the fastest path might be pasting one block into your harness.

In your shell~60 seconds
npm install @secondlayer/sdk

// first query — anonymous, no key
const sl = new SecondLayer();
await sl.index.ftTransfers({ asset: "sbtc" });

# deploy your own view when ready
bun add -g @secondlayer/cli
sl subgraphs deploy my-view.ts
